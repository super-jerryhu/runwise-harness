# Vision

**Runwise: one guy, one team.**

Runwise Harness is a team-grade agent engineering harness for AI coding. It exists because coding agents are becoming powerful enough to participate in real software delivery, but most teams still run them through temporary chats, local sessions, and scattered notes.

The result is useful but hard to manage:

- requirements disappear into chat history
- agents skip critical business questions
- project context is rebuilt over and over
- tests are inconsistent
- risky changes lack workflow gates
- final evidence is hard to audit
- teams cannot see where AI coding work stands

Runwise turns AI coding work into structured engineering runs.

## Product Belief

The next step for AI coding is not only better code generation. It is better engineering control.

Agents should be able to write code, but teams need to know:

- what the agent understood
- what it changed
- what it tested
- what risks were triggered
- what evidence supports completion
- where the work was archived
- what memory should be reused next time

Runwise is designed as the layer that makes this visible and repeatable.

## What Runwise Is

Runwise is:

- a workflow harness for coding agents
- a local-first project context and memory layer
- a requirement run ledger
- a test and verification evidence system
- a final gate for agent work
- a bridge between local agents and team systems such as GitHub, Linear, Jira, Slack, and CI

## What Runwise Is Not

Runwise is not:

- another coding agent
- a generic vector database
- a prompt-only skill collection
- a replacement for Codex, Pi, WorkBuddy, Claude Code, or Cursor Agent
- a pure project management app

Runwise should make existing agents behave more like a reliable engineering teammate.

## Long-Term Goal

The long-term goal is a shared agent engineering workflow standard:

```text
intake -> context -> design -> implementation -> tests -> archive -> memory
```

Different agents should be able to participate in that workflow through adapters, while teams retain consistent state, policy, evidence, and visibility.
